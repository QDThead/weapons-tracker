### 6. NASA FIRMS Satellite Thermal Monitoring (Major Feature)

**Connector:** `src/ingestion/firms_thermal.py`
- Queries NASA FIRMS Area API (VIIRS NOAA-20, 375m resolution) for thermal anomaly detections
- 18 facility bounding boxes with adaptive radius (2km urban refineries, 8km DRC open-pit mines)
- Filters: drops low-confidence detections, drops off-center bush fires (>60% radius from center)
- Includes scan/track pixel dimensions for footprint rendering
- 6-hour in-memory cache, fallback seed data when no API key configured
- Accepts `NASA_FIRMS_MAP_KEY` or `MAP_KEY` env var (free from NASA)

**Status Detection:**
- ACTIVE = 1+ nominal/high-confidence thermal detection in 5 days
- IDLE = 0 detections
- UNKNOWN = API failure or no key

**Globe Integration:**
- Pin outline color = thermal status (green=ACTIVE, amber=IDLE, grey=UNKNOWN)
- Fuzzy red thermal bloom markers (3 concentric translucent red points) at each detection point
- `disableDepthTestDistance: POSITIVE_INFINITY` prevents globe clipping at all zoom levels
- NASA GIBS VIIRS thermal anomaly tile overlay (toggleable WMTS layer, ON by default)
- Two new layer toggles: "VIIRS Thermal (NASA)" and "Thermal Status"
- Legend updated with thermal status key and FIRMS detection marker

**Dossier Popup:**
- "Satellite Verification" section in mine/refinery click popups
- Shows status badge, detection count, latest detection date, brightness (K), FRP (MW)
- IDLE warning message when no thermal signature detected
- Click thermal blob for detection details (brightness, FRP, confidence, timestamp)

**Historical Tracking:**
- 30-day backfill on first scheduler run (108 API calls across 18 facilities)
- Daily snapshots persisted to `data/firms_thermal_history.json` (capped at 90 days)
- **FRP sparkline chart** (Chart.js) in both thermal detection popup and dossier popup
- Shows fire radiative power (MW) trend over 30 days

**Scheduler:**
- Job #27: "NASA FIRMS facility thermal monitoring (18 facilities)" — runs every 6 hours
- Auto-backfills history if `firms_thermal_history.json` doesn't exist
- Snapshots current detections to history on each run

**Tests:** `tests/test_firms_thermal.py` — 10 tests covering facility config, bounding box computation, status logic, and fallback data

## Known Issues / Technical Debt
1. **VIIRS 375m resolution limitation** — detects fire-scale heat, not enclosed industrial operations. Most refineries show 0 detections. Sentinel-5P NO2 would provide better coverage for industrial activity verification.
2. **Ellipse entities clip on globe surface** — all ground-level markers use `disableDepthTestDistance: POSITIVE_INFINITY` as workaround. CesiumJS terrain rendering hides entities below ~200m altitude.
3. **GIBS WMTS tile layer** — renders global thermal anomalies but tiles may be 1-2 days behind NRT. Best used as context overlay, not precision monitoring.
4. **History backfill** — queries 108 API calls on first run. If MAP_KEY rate limited, some dates may be missing.
