# Session Handoff — 2026-04-03 (Session C)

## What Was Done This Session

### 1. Thermal Heat Patches on 3D Globe (Enhancement)
- Replaced fixed-pixel point markers with **ground-hugging ellipse entities** for thermal detections
- 3 concentric ellipses per detection: outer glow (6x footprint), mid bloom (3x), hot core (1.5x VIIRS footprint)
- Sizes derived from actual VIIRS scan/track pixel dimensions (~375m), so patches scale with zoom
- Ellipses use `classificationType: BOTH` and `height: 0` to hug terrain surface
- Small bright-center point (8px) with temperature label stays visible at all zoom levels
- Fixed CesiumJS `semiMajorAxis >= semiMinorAxis` requirement with `Math.max`/`Math.min`

### 2. Per-Flyby FRP Sparklines (Enhancement)
- Changed sparklines from daily-aggregate line charts to **per-flyby bar charts**
- Each bar = one satellite pass, labeled with date + time (e.g., "04-03 0130")
- Bar color scales from orange to red based on FRP intensity
- Tooltips show "Flyby: MM-DD HHMM" and "X.XX MW FRP"
- Falls back to daily aggregates for old history data without per-flyby records
- Applied to both thermal detection popup and facility dossier popup
- Backend (`firms_thermal.py`) now stores per-flyby `{date, time, frp}` records in history
- Backward-compatible: old history entries without `flybys` still work

## Known Issues / Technical Debt
1. **Overland route altitude** — 500m polylines may clip on some terrain. If visible, increase to 1000-2000m.
2. **Route waypoint precision** — Waypoints are approximate (~10-50km accuracy). For higher precision, integrate searoute-js or Eurostat maritime routing for dynamic sea lane calculation.
3. **Non-cobalt minerals** — Still use the old `shipping_routes` key and SEA_ROUTES fallback. Need migration when those minerals get deep-dive treatment.
4. **VIIRS thermal ellipses** — Work well but ground-level ellipses may not render on all terrain providers. The `disableDepthTestDistance: POSITIVE_INFINITY` on the center point ensures the temperature label is always visible.
5. **Per-flyby history** — Old history entries (pre-session) don't have `flybys` field. Frontend falls back gracefully. New snapshots will include per-flyby data.
